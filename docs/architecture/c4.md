# Arquitetura C4 (Context, Containers, Components) 🏛️

Documentação visual da arquitetura do SalesOS Ultimate usando o modelo C4.

## Nível 1: Contexto

O diagrama de contexto mostra como o SalesOS se relaciona com o mundo exterior.

```mermaid
C4Context
    title Diagrama de Contexto do Sistema SalesOS

    Person(customer, "Usuário Final", "Equipes de Vendas, SDRs e Gestores.")
    System(salesos, "SalesOS Ultimate", "Plataforma de CRM e Automação de Vendas.")

    System_Ext(hubspot, "HubSpot CRM", "Sincronização bidirecional de deals e contatos.")
    System_Ext(openai, "OpenAI / Anthropic", "Modelos de LLM para geração de copy e análise.")
    System_Ext(linkedin, "LinkedIn / Google", "Fontes de dados para enriquecimento (via Scraping).")
    System_Ext(stripe, "Stripe", "Processamento de pagamentos.")
    System_Ext(resend, "Resend", "Envio de e-mails transacionais e cold mail.")

    Rel(customer, salesos, "Gerencia leads, campanhas e automações")
    Rel(salesos, hubspot, "Sincroniza dados (Sync)")
    Rel(salesos, openai, "Envia prompts / Recebe completations")
    Rel(salesos, linkedin, "Extrai dados públicos")
    Rel(salesos, stripe, "Gerencia assinaturas")
    Rel(salesos, resend, "Envia e-mails")
```

## Nível 2: Containers

O diagrama de containers mostra as aplicações e serviços executáveis.

```mermaid
C4Container
    title Diagrama de Containers

    Person(user, "Usuário", "Acessa via Browser")

    Container_Boundary(c1, "SalesOS System") {
        Container(web_app, "Web Application", "Next.js 14", "Frontend (React) e BFF. Serve a UI e API Routes.")
        Container(ai_service, "AI Service", "Python / FastAPI", "Microserviço para scraping, RAG e orquestração de Agentes.")
        Container(worker, "Background Workers", "Node.js / BullMQ", "Processamento assíncrono de filas (e-mails, sync CRM).")

        ContainerDb(database, "Database", "PostgreSQL", "Armazena usuários, leads, configurações e vetores (pgvector).")
        ContainerDb(redis, "Cache & Queue", "Redis", "Filas de tarefas e cache de sessão.")
    }

    Rel(user, web_app, "HTTPS / JSON", "Usa")

    Rel(web_app, database, "SQL (Prisma)", "Lê/Escreve")
    Rel(web_app, redis, "Redis Protocol", "Enfileira Jobs")
    Rel(web_app, ai_service, "HTTP / REST", "Solicita análise IA")

    Rel(worker, redis, "Redis Protocol", "Consome Jobs")
    Rel(worker, database, "SQL (Prisma)", "Atualiza status")
    Rel(worker, ai_service, "HTTP", "Aciona agentes")

    Rel(ai_service, database, "SQL (Asyncpg)", "Lê dados / Vetores")
```

## Nível 3: Componentes (Web App)

Visão interna do container `apps/web`.

```mermaid
C4Component
    title Componentes do Web App (Next.js)

    Container(web, "Web App", "Next.js")

    Component(auth, "Auth Module", "NextAuth.js", "Gerencia sessões e JWT.")
    Component(api_routes, "API Routes", "Next.js API", "Endpoints REST para o frontend.")
    Component(server_actions, "Server Actions", "React Server Actions", "Mutations diretas do frontend.")
    Component(hooks, "React Query Hooks", "TanStack Query", "Gerencia estado assíncrono e cache no cliente.")

    Component(prisma, "Prisma Client", "ORM", "Abstração do banco de dados.")

    Rel(api_routes, prisma, "Usa")
    Rel(server_actions, prisma, "Usa")
    Rel(hooks, api_routes, "Fetch")
```

## Decisões Arquiteturais

1. **Separação de Responsabilidades**:
   - **Node.js/Next.js**: Focado em I/O, UI e integrações rápidas.
   - **Python**: Focado em processamento pesado (CPU bound), manipulação de dados e bibliotecas de IA (LangChain, Pandas).

2. **Comunicação Assíncrona**:
   - Operações lentas (ex: "Enriquecer 1000 Leads") são enviadas para o Redis e processadas pelos Workers, liberando a UI imediatamente.

3. **Banco de Dados Compartilhado**:
   - O `ai-agents` acessa o mesmo banco PostgreSQL que o `web`. Isso evita a necessidade de duplicação de dados, mas exige cuidado com migrações (gerenciadas pelo Prisma no lado Node).
